# Space

Hearth Tokens provides a consistent set of spacing values for use in layout and component spacing across your projects. These tokens ensure uniform padding, margins, and gaps.

## Getting Started

To use the space tokens in your project, you can import them from the `@utilitywarehouse/hearth-tokens` package. The tokens are available in multiple formats depending on your needs:

1.  **Raw Values (JS):** Provides raw pixel values. Use this in any JavaScript environment (Node.js, React Native, Browser) when you need the direct numerical value.
2.  **CSS Variable Strings (Browser JS):** Provides strings formatted as CSS variables (`var(--h-space-...)`). Use this in browser JavaScript (React, Vue, etc.) when applying styles directly that leverage CSS variables.
3.  **CSS Definitions:** Provides the actual CSS custom property definitions. This must be imported for the CSS Variable Strings (Browser JS) format to work correctly.

### JS/TS Usage (`/js`)

Import the `space` object from `@utilitywarehouse/hearth-tokens/js/space` to access the raw pixel values directly. This format is universally applicable in any JavaScript environment.

```js
import { space } from '@utilitywarehouse/hearth-tokens/js'; // Note the default export

const smallPadding = space[50]; // 4 (pixels)
const largeMargin = space[400]; // 32 (pixels)

// Example usage in React Native
const styles = StyleSheet.create({
  container: {
    padding: space[200], // 16
    marginTop: space[300], // 24
  },
});

// Example usage in browser JS for calculations or non-CSS contexts
const elementWidth = baseWidth + space[100];
```

### JS/TS CSS Variable String Usage (Browser JS - `/browser`)

Import the `space` object from `@utilitywarehouse/hearth-tokens/browser/space` to access CSS variable strings. This format is specifically designed for browser environments where you intend to use CSS variables within your JavaScript code (e.g., inline styles, CSS-in-JS libraries). **Requires the CSS definitions to be imported separately (see next section).**

```js
import { space } from '@utilitywarehouse/hearth-tokens/browser';

const smallPaddingVar = space[50]; // 'var(--h-space-50)'
const largeMarginVar = space[400]; // 'var(--h-space-400)'

// Example usage in a React component's style prop
const MyComponent = () => (
  <div style={{ padding: space[100], marginBottom: space[200] }}>
    Content using CSS variables via JS
  </div>
);

// Example usage with styled-components
import styled from 'styled-components';

const StyledContainer = styled.div`
  padding: ${space[200]}; /* Resolves to var(--h-space-200) */
  margin-top: ${space[300]}; /* Resolves to var(--h-space-300) */
`;
```

### CSS Usage (`/css`)

Import the `space.css` file in your application's entry point (e.g., main JS/TS file or global CSS file). This file defines the actual CSS custom properties (e.g., `--h-space-100: 8px;`). **This step is necessary for the CSS Variable String (Browser JS) format to function correctly.** It also allows you to use the variables directly in your standard CSS files.

```js
// Import in your main JS/TS file or CSS entry point
import '@utilitywarehouse/hearth-tokens/css/space.css';
```

```css
/* Example usage directly in a CSS file */
.my-component {
  padding: var(--h-space-100); /* 8px */
  margin-bottom: var(--h-space-300); /* 24px */
}

/* Example using CSS Modules or similar */
:root {
  --component-padding: var(--h-space-200); /* 16px */
}

.container {
  padding: var(--component-padding);
}
```

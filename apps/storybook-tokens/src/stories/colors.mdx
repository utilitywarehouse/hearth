## Getting Started

To use the colours in your project, you can import them from the `@utilitywarehouse/hearth-tokens` package. The colours are available in multiple formats depending on your needs:

1.  **Raw Hex Values (JS):** Provides raw hex color strings (e.g., `'#ffffff'`). Use this in any JavaScript environment (Node.js, React Native, Browser) when you need the direct color value.
2.  **CSS Variable Strings (Browser JS):** Provides strings formatted as CSS variables (`var(--h-color-...)`). Use this in browser JavaScript (React, Vue, etc.) when applying styles directly that leverage CSS variables.
3.  **CSS Definitions:** Provides the actual CSS custom property definitions (e.g., `--h-color-blue-500: #41a0ff;`). This must be imported for the CSS Variable Strings (Browser JS) format to work correctly.

### JS/TS Usage (`/js`)

Import the `color` object from `@utilitywarehouse/hearth-tokens/js` to access the raw hex values directly. This format is universally applicable in any JavaScript environment.

```js
import { color } from '@utilitywarehouse/hearth-tokens/js';

// Color scales
const primaryColor = color.blue[500]; // '#41a0ff'
const backgroundGrey = color.grey[50]; // '#f6f5f5'

// Semantic theme colors
const brandBackground = color.light.background.brand; // '#7a42c8'
const primaryText = color.light.text.primary; // '#101010'
const dangerColor = color.light.feedback.danger.default.color; // '#de2612'

// Dark theme equivalents
const darkBrandBackground = color.dark.background.brand; // '#7a42c8'
const darkPrimaryText = color.dark.text.primary; // '#e7e6e6'
const darkDangerColor = color.dark.feedback.danger.default.color; // '#ff7964'

// Example usage in React Native
import { StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  button: {
    backgroundColor: color.light.surface.brand.default.color,
    color: color.light.text.inverted,
  },
  errorText: {
    color: color.light.feedback.danger.default.color,
  },
});

// Example usage in browser JS for non-CSS contexts (e.g., canvas)
// ctx.fillStyle = color.red[500];
```

### JS/TS CSS Variable String Usage (Browser JS - `/browser`)

Import the `color` object from `@utilitywarehouse/hearth-tokens/browser/color` to access CSS variable strings. This format is specifically designed for browser environments where you intend to use CSS variables within your JavaScript code (e.g., inline styles, CSS-in-JS libraries). **Requires the CSS definitions to be imported separately (see next section).**

```js
import { color } from '@utilitywarehouse/hearth-tokens/browser';

// Color scales
const primaryColorVar = color.blue[500]; // 'var(--h-color-blue-500)'
const textColorVar = color.grey[900]; // 'var(--h-color-grey-900)'

// Semantic theme colors
const brandBgVar = color.light.background.brand; // 'var(--h-color-light-background-brand)'
const primaryTextVar = color.light.text.primary; // 'var(--h-color-light-text-primary)'
const dangerColorVar = color.light.feedback.danger.default.color; // 'var(--h-color-light-feedback-danger-default-color)'

// Example usage in a React component's style prop
const MyComponent = () => (
  <div
    style={{
      backgroundColor: color.light.background.primary,
      color: color.light.text.primary,
      borderColor: color.light.border.subtle.color,
    }}
  >
    Hello World
  </div>
);

// Example usage with styled-components
import styled from 'styled-components';

const StyledButton = styled.button`
  background-color: ${color.light.surface.brand.default.color};
  color: ${color.light.text.inverted};
  border: 1px solid ${color.light.border.subtle.color};

  &:hover {
    background-color: ${color.light.interactive.brand.surface.strong.hoverColor};
  }

  &:focus {
    outline: 2px solid ${color.light.focus.focus};
  }
`;

const ErrorMessage = styled.div`
  background-color: ${color.light.feedback.danger.subtle.color};
  color: ${color.light.feedback.danger.default.color};
  border: 1px solid ${color.light.feedback.danger.default.color};
`;
```

### CSS Usage (`/css`)

Import the `color.css` file in your application's entry point (e.g., main JS/TS file or global CSS file). This file defines the actual CSS custom properties (e.g., `--h-color-blue-500: #41a0ff;`). **This step is necessary for the CSS Variable String (Browser JS) format to function correctly.** It also allows you to use the variables directly in your standard CSS files.

```js
// Import in your main JS/TS file or CSS entry point
import '@utilitywarehouse/hearth-tokens/css/color.css';
```

```css
/* Example usage directly in a CSS file */

/* Using color scales */
.my-element {
  background-color: var(--h-color-warm-white-50);
  color: var(--h-color-grey-900);
  border-color: var(--h-color-blue-500);
}

/* Using semantic theme colors */
.button-primary {
  background-color: var(--h-color-light-surface-brand-default-color);
  color: var(--h-color-light-text-inverted);
  border: 1px solid var(--h-color-light-border-subtle-color);
}

.button-primary:hover {
  background-color: var(--h-color-light-interactive-brand-surface-strong-hover-color);
}

.button-primary:focus {
  outline: 2px solid var(--h-color-light-focus-focus);
}

/* Error states */
.error-message {
  background-color: var(--h-color-light-feedback-danger-subtle-color);
  color: var(--h-color-light-feedback-danger-default-color);
  border: 1px solid var(--h-color-light-feedback-danger-default-color);
}

/* Dark theme variant */
.dark .button-primary {
  background-color: var(--h-color-dark-surface-brand-default-color);
  color: var(--h-color-dark-text-inverted);
  border-color: var(--h-color-dark-border-subtle-color);
}

.dark .error-message {
  background-color: var(--h-color-dark-feedback-danger-subtle-color);
  color: var(--h-color-dark-feedback-danger-default-color);
  border-color: var(--h-color-dark-feedback-danger-default-color);
}
```

## Semantic Theme Colors

### What are Semantic Tokens?

Semantic tokens provide meaning-based color assignments that adapt to different themes (light/dark). Instead of using specific color values like `blue[500]`, semantic tokens use names that describe their purpose like `background.brand` or `feedback.danger.default.color`.

**Benefits of semantic tokens:**

- **Theme-aware**: Automatically adapt to light and dark themes
- **Consistent**: Ensure consistent color usage across your application
- **Maintainable**: Change themes without updating individual color references
- **Accessible**: Pre-configured color combinations that meet accessibility standards

### Using Semantic Tokens

Semantic tokens are organized into logical categories:

- **Background**: Page and component background colors
- **Text**: Text colors for different contexts
- **Border**: Border colors with different emphasis levels
- **Focus**: Focus indicator colors
- **Feedback**: Colors for different states (danger, success, warning, info, functional)
- **Interactive**: Colors for interactive elements (buttons, links) with hover/active states
- **Surface**: Colored surfaces for different services and contexts
- **Shadow**: Shadow colors for elevation and depth

```js
// Instead of using specific colors
const badButton = {
  backgroundColor: color.purple[700], // Hard-coded purple
  color: color.white[0], // Hard-coded white
};

// Use semantic tokens
const goodButton = {
  backgroundColor: color.light.surface.brand.default.color, // Semantic brand color
  color: color.light.text.inverted, // Semantic inverted text
};
```

### Theme Switching

With semantic tokens, switching between light and dark themes is straightforward:

```css
/* Light theme (default) */
.button {
  background-color: var(--h-color-light-surface-brand-default-color);
  color: var(--h-color-light-text-inverted);
}

/* Dark theme */
.dark .button {
  background-color: var(--h-color-dark-surface-brand-default-color);
  color: var(--h-color-dark-text-inverted);
}
```

```js
// In JavaScript/React
const Button = ({ theme = 'light' }) => {
  const colors = theme === 'light' ? color.light : color.dark;

  return (
    <button
      style={{
        backgroundColor: colors.surface.brand.default.color,
        color: colors.text.inverted,
        border: `1px solid ${colors.border.subtle.color}`,
      }}
    >
      Click me
    </button>
  );
};
```

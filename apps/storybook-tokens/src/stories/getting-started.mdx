# Getting Started

This guide explains the different ways you can consume our tokens in your projects.

- [Token Formats](#token-formats)
- [Token Types](#token-types)
- [Usage Examples](#usage-examples)

## Token Formats

Hearth Tokens are distributed in several formats to accommodate various development environments and use cases:

1.  **Raw JavaScript Values (`/js`)**

    - **What:** Provides the direct, raw values of the tokens (e.g., hex codes for colours, pixel numbers for spacing).
    - **Path:** `@utilitywarehouse/hearth-tokens/js` or `@utilitywarehouse/hearth-tokens/js/{tokenType}` (e.g., `@utilitywarehouse/hearth-tokens/js/color`, `@utilitywarehouse/hearth-tokens/js/space`)
    - **Use When:** You need the actual value in any JavaScript/TypeScript environment (Node.js, React Native, Browser JS for calculations, etc.).

2.  **Browser JS - CSS Variable Strings (`/browser`)**

    - **What:** Provides JavaScript objects where the values are strings representing CSS variables (e.g., `'var(--h-color-blue-500)'`, `'var(--h-space-100)'`).
    - **Path:** `@utilitywarehouse/hearth-tokens/browser` or `@utilitywarehouse/hearth-tokens/browser/{tokenType}` (e.g., `@utilitywarehouse/hearth-tokens/browser/color`, `@utilitywarehouse/hearth-tokens/browser/space`)
    - **Use When:** You are working in a browser environment (React, Vue, Angular, plain JS) and want to apply styles using CSS variables via JavaScript (e.g., inline styles, CSS-in-JS libraries).
    - **Dependency:** Requires the corresponding CSS Definitions (`/css`) to be imported into your project for the variables to be defined.

3.  **CSS Definitions (`/css`)**
    - **What:** Provides CSS files that define the actual CSS custom properties (e.g., `--h-color-blue-500: #41a0ff;`, `--h-space-100: 8px;`).
    - **Path:** `@utilitywarehouse/hearth-tokens/css` or `@utilitywarehouse/hearth-tokens/css/{tokenType}.css` (e.g., `@utilitywarehouse/hearth-tokens/css/color.css`, `@utilitywarehouse/hearth-tokens/css/space.css`)
    - **Use When:**
      - You are using the Browser JS format (this file _must_ be imported).
      - You want to use the tokens directly within your standard CSS, SCSS, or CSS Modules files.

## Token Types

Hearth provides tokens for various design attributes. Here's a brief overview:

- **Color:** Defines the color palette, including brand colors and common colors, light and dark modes.
- **Color - Semantic:** Defines semantic colors for specific use cases (e.g., success, error, warning). _**Coming Soon!**_
- **Space:** Provides consistent spacing values for margins, paddings, and layout gaps.
- **Font:** Includes font families, sizes, and weights for typography.
- **Line Height:** Defines line heights corresponding to font sizes for optimal readability.
- **Letter Spacing:** Specifies tracking values for text.
- **Border:** Contains tokens for border radius and border width.
- **Shadow:** Defines elevation levels using box shadows.
- **Layout:** Tokens related to layout structures, like container widths (may be less commonly used directly).
- **Primitive:** A single export containing _all_ core token types (Color, Space, Font, Border, etc.) nested within it.
- **Typography:** **(Internal Use)** Responsive, pre-composed typography styles (combining font family, size, weight, line height, etc.) tailored for different breakpoints (mobile, tablet, desktop). These are primarily used internally within Hearth component libraries (React, React Native) and are generally **not intended for direct consumption** in product applications.
- **Components:** **(Internal Use)** These tokens are specifically tailored for building Hearth's internal component libraries (React, React Native). While available, they are generally **not intended for direct consumption** in product applications. Use the core tokens (Color, Space, Font, etc.) instead.

Refer to the specific documentation page for each token type for detailed information and available values.

## Usage Examples

Below are general examples. Refer to the specific documentation for each token type (Colours, Space, etc.) for detailed structures and available tokens.

### Raw JavaScript Values (`/js`)

```javascript
// Example: Using space tokens in React Native
import { StyleSheet } from 'react-native';
import { space } from '@utilitywarehouse/hearth-tokens/js';

const styles = StyleSheet.create({
  container: {
    padding: space[100], // Gets the raw pixel value, e.g., 8
    margin: space[200], // e.g., 16
  },
});

// Example: Using color tokens in a Node.js script
import { color } from '@utilitywarehouse/hearth-tokens/js';

function logPrimaryColor() {
  console.log('Primary color (light):', color.light.blue[500]); // Logs the hex code, e.g., '#41a0ff'
}
```

### CSS Definitions (`/css`)

**Step 1: Import CSS Definitions (do this once in your app)**

```javascript
// Import all CSS definitions
import '@utilitywarehouse/hearth-tokens/css';
```

**Step 2: Use CSS Variables in your CSS/SCSS/CSS Modules**

```css
/* styles.css */
.alert {
  padding: var(--h-space-200);
  background-color: var(--h-color-red-100);
  color: var(--h-color-red-900);
  border: 1px solid var(--h-color-red-300);
  border-radius: var(--h-space-50);
}

.page-title {
  color: var(--h-color-purple-700);
  margin-bottom: var(--h-space-400);
}
```

```css
/* component.module.css */
.container {
  padding: var(--h-space-300);
}

.text {
  color: var(--h-color-grey-700);
}
```

### Browser JS - CSS Variable Strings (`/browser`)

**Step 1: Import CSS Definitions (do this once in your app)**

```javascript
// Import all CSS definitions
import '@utilitywarehouse/hearth-tokens/css';
```

**Step 2: Use Browser JS Tokens**

```javascript
// Example: Using tokens in a React component with inline styles
import React from 'react';
import { color, space } from '@utilitywarehouse/hearth-tokens/browser';

const MyButton = ({ children }) => (
  <button
    style={{
      backgroundColor: color.blue[500], // 'var(--h-color-blue-500)'
      color: color.white, // 'var(--h-color-common-white)'
      padding: `${space[100]} ${space[200]}`, // 'var(--h-space-100) var(--h-space-200)'
      border: 'none',
      borderRadius: space[50], // 'var(--h-space-50)'
    }}
  >
    {children}
  </button>
);

// Example: Using tokens with styled-components
import styled from 'styled-components';
import { color, space } from '@utilitywarehouse/hearth-tokens/browser';

const StyledCard = styled.div`
  background-color: ${color.warmWhite[50]};
  padding: ${space[300]};
  border: 1px solid ${color.grey[200]};
  box-shadow: 0 ${space[25]} ${space[50]} ${color.grey[300]};
`;
```

Choose the format that best suits your project's architecture and the specific
task at hand. Remember to consult the individual token documentation pages for
the complete list of available tokens and their specific values or variable
names.

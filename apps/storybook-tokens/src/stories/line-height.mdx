# Line Height

Hearth Tokens provides a set of line height values to ensure consistent vertical spacing within text elements across your projects. These tokens help maintain readability and visual harmony.

## Getting Started

To use the line height tokens in your project, you can import them from the `@utilitywarehouse/hearth-tokens` package. The tokens are available in multiple formats depending on your needs:

1.  **Raw Values (JS):** Provides raw pixel values. Use this in any JavaScript environment (Node.js, React Native, Browser) when you need the direct numerical value for calculations or specific styling contexts (like React Native).
2.  **CSS Variable Strings (Browser JS):** Provides strings formatted as CSS variables (`var(--h-line-height-...)`). Use this in browser JavaScript (React, Vue, etc.) when applying styles directly that leverage CSS variables.
3.  **CSS Definitions:** Provides the actual CSS custom property definitions (usually in `rem`). This must be imported for the CSS Variable Strings (Browser JS) format to work correctly.

### JS/TS Usage (`/js`)

Import the `lineHeight` object from `@utilitywarehouse/hearth-tokens/js/line-height` to access the raw pixel values directly. This format is universally applicable in any JavaScript environment.

```js
import { lineHeight } from '@utilitywarehouse/hearth-tokens/js';

const bodyLineHeight = lineHeight[300]; // 20 (pixels)
const headingLineHeight = lineHeight[900]; // 40 (pixels)

// Example usage in React Native
import { StyleSheet, Text } from 'react-native';

const styles = StyleSheet.create({
  paragraph: {
    fontSize: 16, // Example font size
    lineHeight: lineHeight[300], // 20
  },
  heading: {
    fontSize: 32, // Example font size
    lineHeight: lineHeight[900], // 40
  },
});

const MyTextComponent = () => (
  <>
    <Text style={styles.heading}>Heading Text</Text>
    <Text style={styles.paragraph}>Paragraph text with appropriate line height.</Text>
  </>
);
```

### JS/TS CSS Variable String Usage (Browser JS - `/browser`)

Import the `lineHeight` object from `@utilitywarehouse/hearth-tokens/browser/line-height` to access CSS variable strings. This format is specifically designed for browser environments where you intend to use CSS variables within your JavaScript code (e.g., inline styles, CSS-in-JS libraries). **Requires the CSS definitions to be imported separately (see next section).**

```js
import { lineHeight } from '@utilitywarehouse/hearth-tokens/browser';

const bodyLineHeightVar = lineHeight[300]; // 'var(--h-line-height-300)'
const headingLineHeightVar = lineHeight[900]; // 'var(--h-line-height-900)'

// Example usage in a React component's style prop
const MyStyledText = ({ variant }) => (
  <p style={{ lineHeight: variant === 'heading' ? headingLineHeightVar : bodyLineHeightVar }}>
    Some text content
  </p>
);

// Example usage with styled-components
import styled from 'styled-components';

const Paragraph = styled.p`
  line-height: ${lineHeight[300]}; /* Resolves to var(--h-line-height-300) */
`;

const Heading = styled.h1`
  line-height: ${lineHeight[900]}; /* Resolves to var(--h-line-height-900) */
`;
```

### CSS Usage (`/css`)

Import the `line-height.css` file in your application's entry point (e.g., main JS/TS file or global CSS file). This file defines the actual CSS custom properties (e.g., `--h-line-height-300: 1.25rem;`). **This step is necessary for the CSS Variable String (Browser JS) format to function correctly.** It also allows you to use the variables directly in your standard CSS files.

```js
// Import in your main JS/TS file or CSS entry point
import '@utilitywarehouse/hearth-tokens/css/line-height.css';
```

```css
/* Example usage directly in a CSS file */
.body-text {
  font-size: 1rem; /* 16px */
  line-height: var(--h-line-height-300); /* 1.25rem (20px) */
}

.title {
  font-size: 2rem; /* 32px */
  line-height: var(--h-line-height-900); /* 2.5rem (40px) */
}

/* Example using CSS Modules */
.paragraph {
  line-height: var(--h-line-height-300);
}
```

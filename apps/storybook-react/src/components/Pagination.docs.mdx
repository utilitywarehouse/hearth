## Many Pages

When there are many pages (more than 8), ellipsis (...) are shown to condense the pagination. The component intelligently shows:
- First and last pages always visible
- When near the start: first 4 pages, ellipsis, last 3 pages
- When in the middle: first page, ellipsis, current page with neighbors, ellipsis, last page
- When near the end: first 3 pages, ellipsis, last 4 pages

## Pagination API

This component is based on the `div` element by default and supports the following common props:

- Margin

| Prop          | Type                           | Default | Description                                                   |
| ------------- | ------------------------------ | ------- | ------------------------------------------------------------- |
| `as`          | `'div' \| 'nav'`               | `'div'` | Changes the rendered element into a semantically appropriate alternative |
| `currentPage` | `number`                       | -       | The current active page number (1-indexed)                    |
| `totalPages`  | `number`                       | -       | The total number of pages                                     |
| `onPageChange`| `(page: number) => void`       | -       | Callback when page changes                                    |
| `condensed`   | `boolean`                      | `false` | Whether to show condensed version with only "Page X of Y"     |
| `skip`        | `boolean`                      | `true`  | Whether to show skip to first/last page buttons               |

## Usage Guidelines

- Handle the `onPageChange` callback to update your data
- The component automatically shows a maximum of 8 items (including ellipses)

### Example

```ts
const [currentPage, setCurrentPage] = useState(5);

const handlePageChange = (newPage: number) => {
  // Tracking, loading new data...
  setCurrentPage(newPage)
}

return (
  <Pagination
    currentPage={currentPage}
    totalPages={100}
    onPageChange={handlePageChange}
  />
);
```

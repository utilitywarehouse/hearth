## Container variants

The Table component supports three container variants:

- **subtle** (default) - Light border
- **emphasis** - Strong border
- **none** - No border, transparent rows, white header

## Component structure

The Table uses a compound component pattern:

- **Table** - The container component with scroll wrapper (requires `containerVariant` prop, defaults to `'subtle'`)
- **TableHeader** - Wrapper for header row (renders `<thead>` with embedded `<tr>`)
- **TableHeaderCell** - Header cell component (renders `<th>`)
- **TableBody** - Wrapper for table body rows (renders `<tbody>`)
- **TableRow** - Row component (renders `<tr>`)
- **TableCell** - Data cell component (renders `<td>`)

```tsx
<Table containerVariant="subtle">
  <TableHeader>
    <TableHeaderCell>Header 1</TableHeaderCell>
    <TableHeaderCell>Header 2</TableHeaderCell>
  </TableHeader>
  <TableBody>
    <TableRow>
      <TableCell>Data 1</TableCell>
      <TableCell>Data 2</TableCell>
    </TableRow>
  </TableBody>
</Table>
```

## Sorting

The Table supports column sorting through the `TableHeader` and `TableHeaderCell` components. To enable sorting:

1. Pass `sortColumn`, `sortDirection`, and `onSort` props to `TableHeader`
2. Add `sortable` and `sortKey` props to sortable `TableHeaderCell` components
3. Manage sort state in your component and handle the sorting logic

```tsx
const [sortConfig, setSortConfig] = useState({
  column: null,
  direction: 'none'
});

const handleSort = (columnKey) => {
  setSortConfig(prev => {
    if (prev.column === columnKey) {
      // Cycle through: none → asc → desc → none
      const nextDirection =
        prev.direction === 'none' ? 'asc' :
        prev.direction === 'asc' ? 'desc' : 'none';
      return { column: columnKey, direction: nextDirection };
    }
    return { column: columnKey, direction: 'asc' };
  });
};

<Table>
  <TableHeader
    sortColumn={sortConfig.column}
    sortDirection={sortConfig.direction}
    onSort={handleSort}
  >
    <TableHeaderCell sortable sortKey="name">Name</TableHeaderCell>
    <TableHeaderCell sortable sortKey="email">Email</TableHeaderCell>
    <TableHeaderCell>Actions</TableHeaderCell> {/* Not sortable */}
  </TableHeader>
  <TableBody>
    {/* ... sorted data rows */}
  </TableBody>
</Table>
```

**Sort indicators:**
- Chevron icons appear next to sortable column headers
- Active sort direction is highlighted
- Clicking cycles through: none → ascending → descending → none

## Usage guidelines

- Use `TableHeader` for column headers
- Use `TableBody` to wrap data rows
- Use `TableRow` for each row of data
- Use `TableCell` for data cells
- Choose appropriate `containerVariant` based on visual hierarchy needs

### Accessibility
- Ensure table has a descriptive caption or aria-label when needed

## Responsive behavior

The Table component automatically adapts to different screen sizes:

- **Desktop**: Full table layout
- **Tablet/Mobile**: Horizontal scrolling with minimum width of 600px

## Components API

### Table API

The main container component that provides context and handles responsive behavior.

| Prop               | Type                                 | Default    | Description                           |
| ------------------ | ------------------------------------ | ---------- | ------------------------------------- |
| `containerVariant` | `'subtle' \| 'emphasis' \| 'none'`   | `'subtle'` | The border style variant of the table |

Additionally supports all standard `<div>` element props.

### TableHeader API

Wrapper component for the table header row. Automatically wraps children in a `<tr>` element. Provides sorting context to child `TableHeaderCell` components.

| Prop            | Type                                 | Default | Description                                      |
| --------------- | ------------------------------------ | ------- | ------------------------------------------------ |
| `sortColumn`    | `string \| null`                     | -       | The currently sorted column key                  |
| `sortDirection` | `'asc' \| 'desc' \| 'none'`          | -       | The current sort direction                       |
| `onSort`        | `(sortKey: string) => void`          | -       | Callback fired when a sortable column is clicked |

Additionally supports all `<thead>` element props.

### TableHeaderCell API

Component for header cells in the table. Can be made sortable by adding `sortable` and `sortKey` props.

| Prop       | Type      | Default | Description                                              |
| ---------- | --------- | ------- | -------------------------------------------------------- |
| `sortable` | `boolean` | `false` | Whether the column is sortable                           |
| `sortKey`  | `string`  | -       | Unique identifier for this column (required if sortable) |

Additionally supports all `<th>` element props.

### TableBody API

Wrapper component for table body rows.

Extends all `<tbody>` element props.

### TableRow API

Component for table rows.

Extends all `<tr>` element props.

### TableCell API

Component for data cells in the table.

Extends all `<td>` element props.

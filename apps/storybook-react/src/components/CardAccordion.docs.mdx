# CardAccordion

Use `CardAccordion` to allow users to expand or collapse individual content
sections, providing a compact way to present a large amount of information
while maintaining a clean and organised interface.

## CardAccordion type

`CardAccordion` can be rendered as one of two types; `single` or `multiple`. The `type` prop is required.

When `type` is set to `single`, only one item can be expanded at once.

<Canvas of={Stories.Single} />

When `type` is set to `multiple`, then multiple items can be expanded at once.

<Canvas of={Stories.Multiple} />

## Collapsible

The `collapsible` prop can be used when the `type` is set to `single`. When
`true` items can be collapsed, allowing all items to be closed at the same
time.

<Canvas of={Stories.Collapsible} />

## Value

The `value` prop differs depending on whether the `type` is set to `single` or
`multiple`. This also affects the `defaultValue` and `onValueChange` props.

When using a `single` type `CardAccordion`, the value is a `string`.

```tsx
<CardAccordion
  type="single"
  defaultValue="item-1"
>
{...}
</CardAccordion>
```

When using a `multiple` type `CardAccordion`, the value is an array of `string` values.

```tsx
<CardAccordion
  type="multiple"
  defaultValue={["item-1", "item-2"]}
>
{...}
</CardAccordion>
```

## CardAccordion items

The `CardAccordionItem` component, in tandem with `CardAccordionContent`, is intended
to cover most use cases, and uses the header & trigger components internally.
However if you need something more custom, you can use the internal subcomponents;
`CardAccordionHeader` & `CardAccordionTrigger`, ensuring you wrap the trigger in a header component.

```tsx
<CardAccordion type="multiple" heading="Custom item headers">
  <CardAccordionItem value='item-1'>
    <CardAccordionHeader>
      <CardAccordionTrigger>
        <Flex width="100%" alignItems="center" justifyContent="between">
          <BodyText weight="semibold">Custom Header</BodyText>
          <Badge size="sm" colorScheme="positive">New</Badge>
        </Flex>
      </CardAccordionTrigger>
    </CardAccordionHeader>
    <CardAccordionContent>{...}</CardAccordionContent>
  </CardAccordionItem>
  {...}
</CardAccordion>
```

## Heading levels

By default the `CardAccordion` heading renders an `h2` element, and each item
renders an `h3` element. If this doesn't follow the structure of your page, you
can use the `headingElement` prop on `CardAccordion` and `CardAccordionItem` to update
these to an appropriate heading level.

## Accessibility

Follows the [Accordion WAI-ARIA design pattern](https://www.w3.org/WAI/ARIA/apg/patterns/accordion).

While `CardAccordion` is meant to present multiple disclosure items, if you have
variable data that could return only a single item, then using a `CardAccordion` is
still OK. If you know ahead of time you only need to show/hide a single item,
then an alternative disclosure pattern should be used.

## API

This component is based on the `div` element and supports the following common props:

- Margin

| Name              | Type                                                            |
| ----------------- | --------------------------------------------------------------- |
| `heading`         | `string`                                                        |
| `headingElement`  | `"h1" \| "h2" \| "h3" \| "h4"`                                  |
| `helperText`      | `string`                                                        |
| `trailingContent` | `ReactNode`                                                     |
| `type`            | `"multiple" \| "single"`                                        |
| `value`           | `"Array<string>" \| "string"`                                   |
| `defaultValue`    | `"Array<string>" \| "string"`                                   |
| `onValueChange`   | `"(value: Array<string>) => void" \| "(value: string) => void"` |
| `collapsible`     | `boolean`                                                       |
| `disabled`        | `boolean`                                                       |

### CardAccordionItem

Used to render a collapsible accordion section.

| Name             | Type                           |
| ---------------- | ------------------------------ |
| `title`          | `string`                       |
| `description`    | `string`                       |
| `headingElement` | `"h1" \| "h2" \| "h3" \| "h4"` |

### CardAccordionContent

Contains the collapsible content for an accordion item.

### CardAccordionHeader

Wraps a `CardAccordionTrigger`. Use the `as` prop to render the appropriate heading level for your page.

This component is based on the `a` element.

| Name | Type                           |
| ---- | ------------------------------ |
| `as` | `"h1" \| "h2" \| "h3" \| "h4"` |

### CardAccordionTrigger

Toggles the collapsed state of its associated item. It should be nested inside a `CardAccordionHeader`.

This component is based on the `button` element.

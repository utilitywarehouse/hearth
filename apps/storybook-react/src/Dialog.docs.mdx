## Usage Guidelines

`Dialog` comprises a number of components you can compose together, and can be
uses as either a controlled or uncontrolled component.

- `DialogRoot` contains all parts of a dialog, providing context to it's children. You must wrap your Dialog with this component.
- `DialogTrigger` wraps a `Button` component and is used to open the dialog. This is not needed for a state controlled dialog.
- `Dialog` is the main component where you should place your content.
- `DialogFooter` is a container for any actions, such as buttons.
- `DialogClose` wraps a `Button` component and is used to close the dialog. This is not needed for a state controlled dialog.

### Uncontrolled usage

```tsx
<DialogRoot>
  <DialogTrigger>
    <Button>Open dialog</Button>
  </DialogTrigger>
  <Dialog
    heading="Hearth dialog"
    description="Overlays content to request a decision or inform users of important information"
  >
    <DialogFooter>
      <DialogClose>
        <Button variant="ghost" colorScheme="grey">
          Cancel
        </Button>
      </DialogClose>
      <DialogClose>
        <Button variant="solid" colorScheme="yellow">
          Primary
        </Button>
      </DialogClose>
    </DialogFooter>
  </Dialog>
</DialogRoot>
```

## API

This component is based on Radix UI's Dialog primitive and supports the following common props:

- Margin

| Prop              | Type      |
| ----------------- | --------- |
| `heading`         | `string`  |
| `description`     | `string`  |
| `hideCloseButton` | `boolean` |

### DialogRoot

The root component that manages the state and accessibility of the dialog.

| Prop           | Type                      | Description                                                                |
| -------------- | ------------------------- | -------------------------------------------------------------------------- |
| `open`         | `boolean`                 | Controls the open state of the dialog                                      |
| `defaultOpen`  | `boolean`                 | Whether the dialog is open by default                                      |
| `onOpenChange` | `(open: boolean) => void` | Callback that fires when the open state changes                            |
| `modal`        | `boolean`                 | Whether the dialog is modal (blocks interaction with the rest of the page) |

### DialogTrigger

A component that triggers the dialog to open when clicked.

### DialogClose

A component that closes the dialog when clicked.

### DialogFooter

A component for dialog actions, typically containing buttons.

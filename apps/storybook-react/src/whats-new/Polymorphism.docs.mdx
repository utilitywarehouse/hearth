# Polymorphism

We are not attempting to recreate the polymorphism of the Material UI
`component` prop. This is tricky, and comes with unnecessary overhead.

Some components will have an `as` prop, which works similarly to the
`component` prop, but with a very specific set of elements it can be set to.

We also include the `asChild` prop where it makes sense, so that you can render
other elements and have the ability to pass appropriate props without the
problems inherent in the previous approach.

```tsx
// render a span element instead of a div
<Box as="span" />

[...]

// render a section element, with token based padding applied
<Flex asChild padding="400">
  <section>{...}</section>
</Flex>

[...]

// render a link that is visually a button, applying element specific props to each element
<Button asChild onClick={handleClick}>
  <a href="https://uw.co.uk/services">View UW services</a>
</Button>
```

You can also use `asChild` as a way of applying small styling changes to
components that don't support specific style props.

```tsx
<Box asChild width="100%" maxWidth="680px">
  <Alert colorScheme="danger">
    {...}
  </Alert>
</Box>
```
